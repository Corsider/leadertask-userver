# Performance Design для leadertask

## 1. Анализ производительности

### Hot Paths
- Получение списка целей пользователя
- Создание целей
- Поиск пользователей

### Медленные операции
- Поиск юзеров (проход с like)

### Требования к производительности
- Время отклика: <40ms для чтения, <100ms для записи
- Пропускная способность >10 RPS

## 2. Стратегия кеширования
Cache-Aside ддля данных пользователей, проверка кеша перед запросом к БД

## 3. Реализация кеширования

### In-memory
Используется lru 

#### Куда добавлен кеш:
